Personal Portfolio v2

A cutting-edge portfolio website featuring liquid glass design, interactive particle systems, and fluid animations — built entirely with vanilla HTML, CSS, and JavaScript.

Status: Live
Year: 2025
Role: Designer & Developer

Overview

When designing a personal portfolio website, the goal is often not just to display past work, but to demonstrate technical ability and creative vision directly through the user experience. The Personal Portfolio v2 was built to accomplish exactly that. Moving away from standard, boring layouts, this website focuses on presenting information through a cutting-edge "liquid glass" aesthetic that balances premium visuals with high performance.

The core concept centers around frosted glass containers that respond dynamically to user input. Using 3D perspective transforms, mouse-tracking hover states, and smooth CSS transitions, the site feels alive and physical. The light-mode palette provides a bright, premium feel, with subtle gradient orbs floating in the background, creating the illusion of deep visual layers.

A major constraint of the project was performance. Many modern portfolio websites that use heavy animations and 3D effects rely on extensive frameworks and massive libraries, which degrade loading speeds and performance on mobile devices. This project serves as a proof of concept that premium, high-fidelity web experiences can be built using zero dependencies—relying entirely on vanilla HTML, CSS, and JS.

Design Philosophy

The design system is based on a modern interpretation of glassmorphism combined with a light, optimistic color palette. Frosted glass cards feature a high level of background blur, translucent borders, and subtle internal glow effects. In addition, we introduced a 3D perspective to the container cards. When the user hovers over a card, the card tilts along its horizontal and vertical axes to follow the mouse, creating a physical sense of depth.

Prismatic light reflections add another layer of visual interest. Rather than using static gradient overlays, we use radial gradients and dynamic masks that shimmer when the cursor moves across the cards, simulating physical light refraction. Soft background gradient orbs move gently, changing positions and colors slightly to create a liquid-like mesh background that responds as the user scrolls.

Key Features

🫧

Liquid Glass Design

Frosted glass cards with high background blur, dynamic drop shadows, and delicate borders that blend seamlessly with the moving background.

✨

Interactive Particles

A high-performance HTML5 Canvas particle system in the hero section that spawns soft, glowing nodes that drift naturally and move away from the user's cursor.

🎭

3D Card Tilt

Cursor-tracking perspective transforms applied dynamically to glass cards. Cards lean towards the mouse position, creating a real 3D depth effect on desktop.

📱

Fully Responsive

Adapts flawlessly to any viewport. The 3D transforms are gracefully disabled on touch devices to ensure the best possible experience on mobile phones.

⚡

Zero Dependencies

Crafted entirely in raw HTML5, CSS3, and ES6 JavaScript. No frameworks, no external animation libraries, ensuring an incredibly small bundle size and instant load times.

🎨

Prismatic Refractions

Subtle rainbow highlights and liquid shimmers that move across the glass borders on hover, reinforcing the liquid glass aesthetic.

Technical Implementation

To achieve the frosted glass look, we rely heavily on modern CSS layout properties like backdrop-filter and complex box-shadow overlays. The 3D perspective is established on the parent containers using CSS custom properties that are modified on the fly by JavaScript event handlers. Below is a code snippet demonstrating the CSS setup for the interactive glass cards: 

.glass-card {
  position: relative;
  background: var(--glass-bg);
  backdrop-filter: var(--glass-blur);
  -webkit-backdrop-filter: var(--glass-blur);
  border: 1px solid var(--glass-border);
  border-radius: var(--radius-lg);
  box-shadow: var(--shadow-glass);
  transition: 
    transform var(--duration-normal) var(--ease-out),
    box-shadow var(--duration-normal) var(--ease-out),
    border-color var(--duration-normal) var(--ease-out);
  transform-style: preserve-3d;
  perspective: 1000px;
  overflow: hidden;
}

.glass-card:hover {
  background: var(--glass-bg-hover);
  border-color: var(--glass-border-hover);
  box-shadow: var(--shadow-glass-hover);
}

.glass-card::before {
  content: '';
  position: absolute;
  top: 0; left: 0; right: 0; bottom: 0;
  background: linear-gradient(
    135deg, 
    rgba(255,255,255,0.4) 0%, 
    rgba(255,255,255,0) 50%, 
    rgba(100,150,255,0.05) 100%
  );
  z-index: 1;
  pointer-events: none;
}

Performance

With multiple complex animations running simultaneously, optimizing for performance was key to maintaining a smooth 60fps refresh rate:

  • IntersectionObserver API: Used to run scroll-reveal animations only when elements enter the viewport, reducing off-screen styling computations.
  • Canvas Optimization: The background particle system uses a lightweight canvas. Animation loops are managed with requestAnimationFrame and automatically pause when the tab goes background or when the hero section scrolls out of view.
  • Passive Event Listeners: Mouse and scroll listeners utilize the { passive: true } option to ensure they never block default browser scrolling behavior.
  • CSS Hardware Acceleration: Smooth transforms rely on translate3d to trigger GPU acceleration, preventing layout reflows.
